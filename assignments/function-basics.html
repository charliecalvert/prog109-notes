Here are some hints. First, be sure all your code is inside the window.onload method:

                
window.onload = () => {
    // All code goes in here.
}

The onload function is called when an HTML file (document) has finished loading into the main browser window. It is important because our JavaScript file uses document.getElementById to retrieve a handle to HTML elements on the web page you created. If the HTML page (document) has not been loaded, it can't be queried with getElementById. It would be like a student in the classroom asking me a question before I had arrived: it just wouldn't work. You have to wait for me to get there, and get set up, and then you can ask me a question. The onload event tells you that the page is loaded and can be queried, it tells you that Charlie is in the classroom and ready to answer questions

Please remember that the window object is only available in the browser. If we try to run code that uses window from the command prompt it will raise an exception. It will, however, run smoothly in a browser.

Once we introduce window into a JavaScript file then we may not be able to run it with Node at the command prompt. I say "may" because we could reference the window object inside a function that does not get called, and hence the rules of scoping would ensure that no exception would be raised. But for now, it is simplest if you just think: "Ok, I have referenced the window object so I should begin working in the browser rather than at the command prompt."

Modify Paragraph Elements

Get a handle to the elements on the page that we are going to need to access. We do this by calling document.getElementById. We use this method to find the HTML paragraph elements on our HTML page so that we can modify their content. 


const simpleFunctionDisplay = document.getElementById('simpleFunctionDisplay');
const functionCallsAction = document.getElementById('functionCallsAction');

Our goal now is to make the text in our functions to appear not only in the console window, but also in our HTML pages. Modify your version of the simple function, so that it will be able to modify the HTML by accessing the textContent property of the paragraph element. This will set the content of the P element. In other words, your goal is to display text in one of the empty paragraph elements on the HTML page. For example, your code might look a little, but not exactly, like this: 


function simple() {
    console.log('simple');
    simpleFunctionDisplay.textContent = "simple";
}

HINT. The function I'm refering to is discussed in the Build Functions section and begins with the word simple, but is a bit longer.

NOTE: We have a slight preference for textContent over innerHTML as it is faster and a bit safer. But if you want to insert HTML into an element rather than plain text, then use innerHTML.

Handle Button Click

Finally, tell the button to call our function when it is clicked


functionCallsAction.onclick = function() {
    simple();
}

Note that we will need to modify the other functions so that, in addition to the console.log statements, they also set the textContent of each paragraph in function-basics.html.

Note also that we don't call a function when we assign it to onclick. In other words, we write this: onclick = simple not onclick = simple(). The latter has side effects that you probably don't want in that it executes the simple function at the time of the assignment, rather than when the button is clicked.

Let's make the program a bit more interesting by allowing the user to enter text, and to insert the text into our list when the button is clicked.

Add a label and an input control to our HTML file:

<label for="nameDisplay">Enter a name</label>
<input type="text" id="nameDisplay" name="nameDisplay">

Notice how we connect the label to the input control via the for attribute of the label and the id attribute of the input element.

We get data from the input like this:

const data = nameDisplay.value;

Add to List Background

Sometimes we have files that we use in multiple places in our code. One such a file is elf-code.js from the ListButton assignment. As you may know, here is the contents of that file:

const elfCode = {
    appendToList: (list, value) => {
        const li = document.createElement("li");
        li.appendChild(document.createTextNode(value));
        list.appendChild(li);
    }
}

We also need to remember how button onclick events work:

const someAction = document.getElementById('someAction');
someAction.onclick = function() {
    // Handle the onclick event in here.
}
