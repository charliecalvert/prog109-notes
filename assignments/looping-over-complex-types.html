Looping over Complex Types

The simple types are number, string, boolean, null, and undefined. These types are essential to JavaScript programming and you must understand them.

There are two important complex types that you must know. The good news is that they are not really that complex, they are not really that hard to understand. The types are:

  • Arrays
  • Objects

Both these types are commonly used in loops, so we will cover that subject at the same time.

Get Started

Arrays

Here is how to declare an array:

    const myArray = [];

We can also pre-populate our array with content:

    const numbers = [1, 2, 3, 4, 5, 8, 10];

Here is how to access a memeber of this array using zero based indexers

    const numbers = [1, 2, 3, 4, 5, 8, 10];
    console.log(numbers[1]);
    console.log(numbers[0]);
    console.log(numbers[numbers.length - 1]);

Here is how to dynamically add an item to an array:

    const numbers = [1, 2, 3, 4, 5, 8, 10];
    numbers.push(12);
    console.log(numbers);
    numbers.pop(); // Remove the last item
    console.log(numbers);

As you will see later, you can put numbers, strings, booleans, arrays, and objects in inside an array. For instance:

    const items = [1, 'two', [3, 4], false, null];

Loop over Arrays

Here is how to loop over an array:

    const items = [1, 'two', [3, 4], false, null];
    for (let i = 0; i < items.length; i++) {
        console.log(items[i]);
    }

Here is a simpler way to loop over an array:

    const numbers = [1, 2, 3, 4, 5, 8, 10];
    for (let number of numbers) {
        console.log(numbers);
    }

Here are JavaScript methods for converting a string to an array and vice-versa:

    function arrayToString(array) {
        console.log(array.join(', '));
    }

    function stringToArray(input) {
        console.log(input.split(','));
    }


    arrayToString(["bundle.js", "*.js.map", ".c9"]);

    stringToArray("bundle.js, *.js.map, .c9");

Objects

Objects contain properties and functions. For now, we will only add properties to our objects.

Here is how to declare an object:

    const myObject = {};

Here is how to declare an object with two properties:

    const myObject = {
        five: 5,
        six: 'six'
    };

Here is how to access a member of this object

    const myObject = {
        five: 5,
        six: 'six'
    };            

    console.log(myObject.five);
    console.log(myObject.six);

Here is how to dynamically add an item to an object:

    const myObject = {
        five: 5,
        six: 'six'
    };            

    myObject.newItem = 'This is a new item in my object.'
    console.log(myObject.five);
    console.log(myObject.six);
    console.log(myObject.newItem);

Global Objects

There are no instructions in this section, it is just a rather lengthy discussion of an important topic.

There are differences between the JavaScript in Node and JavaScript in a browser. Even though Node is built with the same V8 Engine used to create the JavaScript in a browser, there are still differences.

Step back and think for a second. When we are working at the command line there usually are no windows. That is the big difference between a GUI environment and a command-line environment: The GUI has windows and the command line does not. (I'm talking about windows in an application, not about the Microsoft product called Windows.)

As a result, there is no window object at the command prompt and Node does not have a global object called window.. This makes sense because usually there can be no buttons, no input controls, etc at the command prompt.

Once we start using window.onload or document.getElementId then we are committed to using a browser. This is because the window and document objects are only available in a browser.

There are ways to divide programs up so we can put GUI elements in one JavaScript file and non-GUI code in another file. This would allow us to test some of our code at the command line. For now, however, that is beyond the scope of this course.

Node has an object called global that plays the same role as the window object in a browser, but with all the GUI related code stripped from it:

https://developer.mozilla.org/en-US/docs/Glossary/Global_object
