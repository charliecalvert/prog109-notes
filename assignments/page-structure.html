The Primary Sections

Use HTML5 semantic elements to create three main blocks in the index.html page of your theme. Each of the additional pages of your theme, including the About page, should contain these elements.

This page is an example of a document with these three blocks. Each of the major blocks has its own background color. (In this document, also provide a background color for code, but that is a different subject.)

Header Place the main title here
Main Put the primary content, the body of the article, here.
Footer Put the footer, and links to home and/or about, here.

Keep your CSS simple. Define only the rules and declarations I ask you to define. I realize that we are still not creating beautiful pages, but I want us to build up our knowledge bit by bit.

The Header

We create this section with the <header> tag.

In the CSS, define the following properties for this element:

  • background-color
  • text-align

Define a special p tag for the teaser


    header p {
        font-weight: bold;
        color: blue;
    }

Define a class called byline for your attribution:

<p class="byline">by Some Student</p>

by Some Student

In your css rule for this class, define the following declarations.

  • font-style
  • font-family
  • font-size

Main

Use the <main> element for this section. This is the main section of your document, the place where you put most of your content.

In your CSS rule for this element, include only a background-color declaration.

Footer

Use the <footer> tag. In this section, include a copyright (&copy;) notice and a crude navigation section defined as an unordered list. The elements in the list should be hyperlinks leading to places like Home and About.

In the CSS, define the following properties for this element:

  • background-color
  • text-align

Other

We want to include a small image like the information icon to the left. Our goal is to have the text wrap around it so the image is integrated into the page. In particular, we will have at least one relatively small image float left or right. To achieve your goal, wrap it in a figure element with a class called left. Include a figcaption element. This involves putting together about four lines of code. Examples of the four lines are present in the links found in this outline of the code I want you to write.

  1. Declare a figure element with a class of right or left
  2. Inside it put an img element with src set to something about 100px X 100px
  3. Next comes the figcaption
  4. Close the figure element.

Hint: Try to find your own unique image, but if you are stuck, consider using this SVG image

To float left:


    figure.left {
        float: left;
    }

Also use the doo-hickey in the developer tools or your phone to see that your site looks at least reasonable on a mobile device.

I may define a few more CSS rules that you can create for your theme documents.

In this document, I added a second class to my figure element:


    figure.border {
        background-color: lightslategray;
        border: black solid thin;
        padding: 5px;
    }

Clear Floats

You can use the clear property on an element to make it move below and to the left of the image you have floated to the left or right. But you might not need to do that in this case.

If we float an image to the left, then sometimes everything works out right because the text we want to wrap around the image completely surrounds the image. However, there are frequent times when the image sticks out beneath the text that wanted to surround it. In that case, we can hit a but where the next section or paragraph of our content also starts to wrap around our image. For instance, a footer might begin to wrap around the image. To fix this, we could use the following CSS on the footer:


    footer {
        clear: both;
    }
This should ensure that the footer appears beneath the image. Sometimes our CSS code might require that we put a CLASS or ID on a section, and pick out that section in our CSS like this: 

    section.clear {
        clear: both
    }

In this case, I'm assuming that one or more of your sections have their class set to clear:

                    
    <section class="clear"></section>

Turn it in

To turn it in, just push your repository. I'll look in your root directory for your work on your Home page (index.html). Optionally, you can also create a theme directory and put a similar index.html in it, but don't include the same hyperlinks (anchors) you used on Home page.

I'm looking for at least two HTML files and one CSS file.

  • index.html
  • about.html
  • styles.css

The HTML files should all have the three part structure outlined in this document. In the footer, the two HTML files should link to one another with an anchor element embedded in an unordered list. You would create the element like this with emmet:

ul>li>a

Like this:

    <ul>
        <li><a href=""></a></li>
    </ul>

In the code shown above, you need to complete the href attribute and the content of the anchor tag. The content will probably be either Home or About.

Tips

Remember:

  • Include a DOCTYPE and HTML, HEAD, TITLE, and BODY tags in all your HTML documents
  • Properly indent and format your code, then validate with the HTML Validator.
  • Don't use inline styles, instead put styles in CSS files.
  • Use lowercase markup. Not <P>. Instead: <p>
  • Put the various sections of your document in appropriate tags such as HEADER, MAIN or FOOTER.
