Making Decisions

Learn how to use boolean statements to branch from one option to another. If there are two possible paths, we can decide which path to take in a particular set of circumstances. If the our program is in a particular then we do one thing, if it is in another state then we do something else.

Branching and Conditional Statements

When we write code we often need to test if a condition is true or false in order to make a decision. In JavaScript we have three conditional statements:

  • if statement - test if a condition is true or false. Execute code only if it is true.
  • if...else statement - test is a condition is true or false and do one thing if it is true and another if it is false.
  • switch statement;- Sometimes length if-else statements can be hard to read. switch statements provide an alternative syntax.

You will be asked to test and create some code in this assignment. To get started, create a file called javascript-tests/branching.js. Do your work in this file.

if Statement

    const height = 5;
    if (height >= 5) {
        console.log('We have a big one');
    }

if-else Statement

We can have one or more else statements.

    const height = 5;
    if (height >= 5) {
        console.log('We have a big one');
    } else {
        console.log('We have a little one');
    }

More complex:

    const height = 5;
    if (height >= 5) {
        console.log('We have a big one');
    } else if (height >= 0) {
        console.log('We have a little one');
    } else {
        console.log('We have a negative number or invalid number')
    }

Switch statements

    height = 'little';

    switch(height) {
        case 'little':
            console.log('Switch: a little one is smaller than five');
            break;
    
        case ('big'):
            console.log('Switch: a big one is five or greater');
            break;
    
        default:
            console.log('We do not know about this one.')
    }

    window.onload = () => {
        const simpleBranchAction = document.getElementById('simpleBranchAction');
        const simpleBranchDisplay = document.getElementById('simpleBranchDisplay');
        
        simpleBranchAction.onclick = (event) => {
            let height = 4;

                simpleBranchDisplay.textContent = 'We have a big one';
            } else {
                simpleBranchDisplay.textContent = 'We have a little one';
            }   
        }
    }

Use console.log for Debugging

There are two main reasons for using console.log:

  1. When we are writing browser-based applications we use it to help us debug our applications. For instance, if we want to know if an onclick event gets called, we can put concole.log inside the event to see let us know that it was executed:
simpleBranchAction.onclick = (event) => {
        console.log('SIMPLE BRANCH ACTION ON CLICK WAS CALLED');
        let height = 4;
        // AND SO ON
}

Remember that we need to open the Developer Tools and turn to the Console page to see the output.

  1. The other reason to use console.log is when we work at the command line, and then console.log can be a primary means of creating output for our program.
